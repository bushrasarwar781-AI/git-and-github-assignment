## Merge vs Rebase

### Merge
- Combines two branches using a merge commit
- Keeps complete branch history as it is
- Easy and safe for team collaboration

### Rebase
- Moves commits on top of another branch
- Creates a linear (straight) history
- Rewrites commit history, so use carefully

### Key Difference:
Merge preserves history, while rebase creates a clean linear history.
## How I used Git in this project

- Created multiple branches (master, bug-fix, feature-readme)
- Used Pull Requests for merging changes
- Resolved merge conflicts between branches
- Practiced rebase to maintain clean history
